# Slide Maker

웹 챗봇에서 기획한 슬라이드 개요와 참고 이미지를 Codex에 전달하면, 같은 스타일의 완성형 슬라이드 이미지들을 만드는 워크플로입니다. PowerPoint 파일이 아니라 글자가 포함된 슬라이드 이미지가 결과물입니다.

기본 결과는 현대적인 편집 일러스트 스타일입니다. 각 슬라이드는 큰 제목과 2~4개의 짧은 핵심 구절을 사용하며, 실제 사진은 사용자가 제공하고 사용을 지정한 경우에만 적용합니다.

## 사용자가 알면 되는 폴더

```text
Slide Maker/
├─ 넣을곳/       # 참고 슬라이드, 로고, 사진을 전부 여기에 넣기
├─ 결과물/       # 가장 최근에 만든 슬라이드 이미지 가져가기
└─ 지난결과/     # 이전 생성 결과가 날짜와 제목별로 자동 보관됨
```

사용자는 위 세 폴더만 알면 됩니다. 개요, 프롬프트와 진행 기록은 Codex가 `.slide-maker/`에서 관리합니다.

## 빠른 사용법

1. ChatGPT, Gemini 등 원하는 웹 챗봇과 발표 내용을 기획합니다.
2. [`SLIDE_FORMAT.md`](SLIDE_FORMAT.md)를 첨부하거나 복사한 뒤, 대화를 이 형식으로 정리해 달라고 요청합니다.
3. 참고할 PPT 캡처, 로고, 제품 사진 등이 있으면 종류를 구분하지 말고 `넣을곳/`에 넣습니다.
4. 웹 챗봇이 출력한 Markdown 전체를 이 저장소를 연 Codex 채팅에 붙여넣고 생성을 요청합니다.
5. 완성된 이미지는 `결과물/` 바로 아래에서 가져갑니다.

Codex는 PPT 캡처와 무드보드는 디자인 참고 자료로 판단합니다. 실제 사진은 개요에서 `provided_photo`로 지정하거나 사용자가 사용을 요청한 경우에만 결과에 적용합니다. 사진이 없으면 비슷한 사진을 새로 만들지 않고 일러스트로 표현합니다.

## 결과 보관 방식

새로운 전체 생성 요청이 들어오면 Codex는 기존 `결과물/`의 이미지를 먼저 `지난결과/YYYY-MM-DD_HHMM-short-title/`로 옮깁니다. 같은 이름이 있으면 `-02`, `-03`을 붙입니다.

따라서 `결과물/`에는 항상 가장 최근 결과만 있고, 이전 결과는 명령별로 섞이지 않은 채 `지난결과/`에 남습니다. 특정 결과의 수정이나 이어서 만들기는 사용자가 지정한 기존 결과를 그대로 사용합니다.

## 전체 저장소 구조

```text
.
├─ AGENTS.md
├─ README.md
├─ PROJECT_NOTES.md
├─ SLIDE_FORMAT.md
├─ WORKFLOW.md
├─ STYLE_GUIDE.md
├─ 넣을곳/
├─ 결과물/
├─ 지난결과/
└─ .slide-maker/           # Codex 내부 작업 기록
   ├─ current-deck.md
   ├─ current-run.md
   └─ runs/
```

## 필요한 조건

- 사용자는 자신의 계정으로 Codex에 로그인되어 있어야 합니다.
- 해당 Codex 환경에서 이미지 생성 기능을 사용할 수 있어야 합니다.
- 이미지 생성 한도, 동시 처리 수, 실패율은 계정과 환경에 따라 달라질 수 있습니다.

모든 환경에서 20~30장을 한 번에 완성한다고 보장하지 않습니다. 중단되거나 실패한 슬라이드는 내부 진행 기록을 기준으로 이어서 생성합니다.
